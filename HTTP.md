# HTTP 기본 개념

## HTTP / HTTPS

- HTTP는 데이터를 평문 형태로 전송하기 때문에 구현과 운영이 단순하지만 데이터 탈취의 위험성이 존재합니다

- HTTPS는 데이터 암호화가 추가된 HTTP입니다. 그래서 보안 수준이 높습니다.

## 요청/응답 구조

요청 구조와 응답 구조는 브라우저가 서버에게 요청을 보낼 때 쓰는 구조입니다.
요청 구조와 응답 구조 둘다 시작줄, 헤더, 빈줄, 바디로 나뉘어집니다.

시작줄은 HTTP 메소드, HTTP 버전 등 상태를 간략하게 나타냅니다.

헤더는 주로 요청이나 응답에 대한 추가 정보를 담고 있습니다.

빈줄은 헤더와 바디를 구분하기 위해 쓰입니다.

바디는 전송하는 데이터를 담고 있습니다.

## HTTP Method

HTTP Method는 브라우저가 서버에게 요청을 보낼 떄 사용합니다. 각 유형에 따라 수행하는 작업 유형이 정해집니다.

먼저 `get` 메소드는 데이터를 요청하는 역할을 합니다.

`POST` 메소드는 새로운 데이터를 생성하는 역할을 하고
반대로 `PUT` 메소드는 기존 데이터를 수정 및 교체하는 역할을 합니다.

마지막으로 `DELETE` 메소드는 말 그대로 데이터를 삭제하는 역할을 합니다.

# HTTP의 상태 코드

서버는 요청받은 파일을 찾고 먼저 검색 상태의 대한 정보를 클라이언트에게 보냅니다. 이 정보를 상태 코드라 합니다.

먼저 200은 성공이란 뜻으로 요청대로 잘 처리됐을 떄 표시됩니다.
201은 요청이 성공되어서 새로운 리소스가 생성되었을때 표시됩니다.

400은 요청에 문제가 있어 서버가 처리할 수 없을 떄 표시됩니다.

401과 403은 각각 인증되지 않았을 떄와 인증은 되었지만 데이터에 접근할 권한이 없을 때 표시됩니다.

404는 요청한 곳에 해당하는 리소스가 없을 때 표시됩니다.

500은 서버가 에러가 났을 떄 표시 됩니다.
