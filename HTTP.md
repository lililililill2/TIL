# HTTP

HTTP(HyperText Transfer Protocol)는 하이퍼텍스트를 전송하는 약속이라는 의미입니다.

브라우저 주소창에 URL, 주소를 입력하면 브라우저가 서버에게 요청을 하고 서버는 그 요청을 받고 처리합니다. 이런 요청을 주고받는 과정이 전부 HTTP를 통해서 이루어집니다.

# HTTP의 Stateless 구조와 클라이언트-서버 구조

HTTP은 아까 말했던 정보를 주고받는 요청 간에 정보를 유지하지 않는다는 의미에서 상태 비저장(Stateless) 프로토콜이라고 불립니다.

stateless의 장점은 이전 요청을 기억할 필요가 없기 때문에 메모리 부담이 줄어듭니다.  
또 요청을 기억하지 않기 때문에 요청에 로그인 정보 같은 것들을 보내야 합니다. 그래서 다른 서버에서도 같은 형식으로 처리할 수 있기 때문에 서버 확장을 쉽게 할 수 있습니다.

단점으론 매 요청마다 정보를 다 보내야 해서 비효율적입니다.  
또 사용자 상태 관리를 서버가 기억하지 못하기 때문에 따로 기억할 쿠키, 토큰 등으로 처리해야 해서 복잡합니다.

그래서 stateless 구조를 사용하는 큰 이유는 확장성과 서버의 효율 때문입니다.

클라이언트-서버 구조는 서비스를 요청하는 클라이언트, 즉 브라우저와 그 요청을 받는 서버가 분리된 구조입니다.

# 주요 HTTP 메소드

브라우저가 서버에게 요청을 보낼 때에는 여러 유형이 있습니다.  
각 유형에 따라 수행하려는 작업 유형이 결정됩니다.

주요 HTTP 메서드엔

- `GET`
- `POST`
- `PUT`
- `PATCH`
- `DELETE`

등이 있습니다.

먼저 `GET` 메소드는 데이터를 조회하기 위해 이 메소드를 사용할 수 있고,

`POST` 메소드는 새로운 데이터를 생성할 때 사용하고,

`PUT` 메소드는 반대로 기존 데이터를 수정 및 교체할 때 사용합니다.

`PATCH` 메소드는 `PUT`처럼 전체를 수정하는 게 아닌 데이터의 일부만 수정할 때 사용합니다.

마지막으로 `DELETE` 메소드는 말 그대로 데이터를 삭제할 때 사용합니다.

# HTTP의 상태 코드

서버는 클라이언트의 요청을 처리한 뒤, 그 결과에 대한 상태 정보를 클라이언트에게 보냅니다. 이 정보를 상태 코드라 합니다.

대표적으로 `2xx`, `3xx`, `4xx`, `5xx` 등이 있습니다.

먼저 `2xx`은 요청 성공이란 뜻으로 요청대로 잘 처리됐을 때 표시됩니다.

- `200` 정상 성공
- `201` 생성 성공

`3xx`는 요청을 받고 다른 위치에서 처리해야 할 때 표시됩니다. 주로 페이지 이동할 때 사용됩니다.

- `301` 영구 이동
- `302` 임시 이동

다음은 `4xx`로 요청이 잘못되었을 때 표시됩니다.

- `401` 인증이 되지 않음
- `404` 요청한 자원 없음

마지막으로 `5xx`은 서버가 에러가 났을 때 표시됩니다.

- `502` 서버끼리의 문제가 일어남
- `500` 서버 에러

# HTTP 요청과 응답 구조

먼저 요청은 아까 말한 클라이언트가 서버에 요청을 보낼 때 사용합니다. 반대로 응답은 요청받은 결과를 보낼 때 사용합니다.

## 요청 구조

요청 구조는 크게 4가지로 나뉩니다.

- 먼저 시작줄은 메소드, HTTP의 버전 등 핵심 내용들이 들어갑니다.

- 헤더 부분은 실제 데이터를 보내기 전에 요청을 보내는 서버의 주소, 어떤 브라우저인지, 어떤 형식으로 보내는지 등  
  이 데이터가 무슨 데이터인지 설명하는 메타 정보를 담고 있습니다.

- 다음으로 헤더와 바디를 구분하기 위해 한 줄을 띄웁니다.

- 마지막으로 바디는 서버로 보낼 실제 데이터를 보냅니다.  
  주로 로그인할 때 쓰는 id, 비밀번호 같은 사용자가 쓴 내용들을 서버로 보냅니다.

## 응답 구조

응답 구조도 요청 구조와 같은 4가지 방식으로 구성됩니다.

- 먼저 시작줄은 전에 설명한 HTTP 상태 코드로 결과 상태를 알려줍니다.

- 헤더 부분엔 응답 부분의 정보를 알려줍니다.  
  응답 부분의 body에 담긴 데이터의 타입 등을 알려줍니다.

- 요청 구조와 같이 헤더 바디 부분을 구분하기 위해 빈 줄을 사용합니다.

- 마지막으로 서버가 클라이언트에게 돌려주는 실제 코드를  
  body에 넣습니다. HTML 코드나 JSON 코드를 사용합니다.

# 헤더 부분과 바디 부분

구조에서 보이듯이 헤더 부분은 이 요청이나 응답의 데이터가 어떤 정보를 가지고 있는지 설명하는 메타 정보입니다.

예를 들면 택배의 송장이 박스 안의 내용물이 무엇이고 어디로 가는지 알려주는 역할을 합니다.

여기서 헤더가 알려주는 데이터 부분을 바로 바디 부분이라 합니다.  
요청이나 응답에서 실제로 보낼 데이터를 담아두는 역할을 합니다.
