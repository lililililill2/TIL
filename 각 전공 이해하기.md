# 각 전공의 역할과 하는 일

## 프론트엔드

프론트엔드란 사용자가 직접 보고 상호작용하는 웹사이트나 앱의 화면을 사용자가 사용하기 쉽게 만드는 역할을 합니다.

프론트엔드의 기술들 입니다.

- HTML: 웹 페이지의 구조를 담당합니다. 웹의 뼈 역할을 합니다
- CSS: 웹 페이지의 디자인을 담당합니다. 웹의 피부 역할을 합니다
- JavaScript: 사용자와의 상호작용 및 동적인 기능을 담당합니다. 웹의 움직임 역할을 합니다.
- React: UI를 컴포넌트 단위로 구성하여 효율적으로 관리할 수 있게 해주는 라이브러리입니다. 예를 들면 공장처럼 조립하는 느낌입니다.

프론트엔드가 주로 하는 일은 디자이너가 만든 디자인을 실제 웹사이트의 화면으로 구현하며 버튼 클릭시의 애니메이션, 데이터 입력 폼, 그리고 반응형 디자인을 통해 스마트폰이나 컴퓨터 같은 다른 화면 크기에서도 디자인이 깨지지 않게 요소를 구성하여 사용자가 편리하고 쉽게 이용할 수 있는 인터페이스와 경험을 제공합니다.

또 사용자가 요청을 하면 그 요청을 백엔드로 요청을 보내는 시스템을 만듭니다.

## 백엔드

프론트엔드가 사용자들에게 보이는 부분이라면 백엔드는 유저들에게 보이지 않게 시스템의 규칙을 설정하고 사용자의 요청을 받는 역할을 합니다.

백엔드가 주로 하는일은 프론트엔드에서 보내온 정보를 데이터베이스에 저장하거나 처리하고, 사용자가 요청한 정보를 데이터베이스에서 찾아 전달하는 시스템을 만듭니다.
또 서비스의 비즈니스 로직을 설정합니다.

비즈니스 로직이란 제공하는 서비스가 어떻게 동작해야 할지에 대한 의사결정의 과정을 말합니다.

예를 들어 사용자가 돈이 부족하면 결제버튼을 눌러도 결제가 되지 않도록 규칙을 만들 수도 있습니다. 첫번째로 사용자의 잔액이 결제 금액보다 많은가? 다음으로 상품의 재고가 남아 있는가? 등 이런 의사결정 과정을 거쳐 비즈니스 로직이 작동합니다.

## 디자인

디자인의 역할은 사용자가 서비스를 이용할 때 불편한 점이 없도록 사용자의 경험을 분석해 요소의 구조를 설계하고 요소를 꾸며 사용자가 보기 좋게 만드는 역할을 합니다.

디자인이 하는 일은 먼저 사용자가 어떤 상황에서 어떤 행동을 하는지 분석해서 가장 편하게 쓸 수 있는 방법이 무엇인지 분석하고, 설계도를 짠 뒤 보기 좋도록 ui 디자인을 하고,
프론트엔드 개발자가 쉽게 코드로 옮길 수 있도록 색깔, 크기,이미지 등 디자인 요소의 정보를 정리하고 전달하는 일을 합니다.

## 클라우드

클라우드의 역할은 사용자에게 서비스가 끊임없이 지속될 수 있도록 서버 인프라를 구축하고 관리하는 역할을 합니다.

먼저 클라우드란 인터넷을 통해 가상의 서버, 저장소 등을 빌려 쓰는 서비스를 의미하는데 이걸 통해 클라우드는 웹에 많은 인원이 들어오면 서버를 확장하고 적은 인원이 있다면 서버를 축소하는 일과 빠르게 서버를 구축 하는 일을 할 수 있습니다.

# 프론트엔드와 백엔드의 호환

프론트엔드와 백엔드는 서로 다른 환경에서 작동합니다.

그래서 그 둘을 통신하기 위해 어떤 방법을 쓰는데 바로
API(Application Programming Interface)를 사용합니다.
API란 한 프로그램에서 다른 프로그램으로 데이터를 주고받기 위한 방법입니다.

이 API를 활용해서 호환하는데 과정은 크게 4가지로 나뉩니다.

- 먼저 사용자가 화면에서 특정 동작으로 요청을 하면 프론트엔드는 그 정보를 담아서 백엔드로 보냅니다 이때 API를 사용합니다

- 다음으로 백엔드는 그 정보를 확인하고, 그 정보를 저장하거나 처리하고 정해놓은 규칙에 따라 그 정보의 결과를 계산합니다.

- 처리가 끝나면 백엔드는 그 결과를 JSON이라는 텍스트 형식으로 정리하여 프론트엔드에 다시 보냅니다. 이때도 API를 사용합니다

- 마지막으로 프론트엔드는 전달받은 JSON 텍스트를 바탕으로
  사용자 화면에 처리한 결과를 전달합니다

비유를 하면 사람을 프론트엔드, 차키를 API, 자동차는 백엔드로 생각 할 수 있습니다.

사람이 그냥 말로 차문 열려라 하면 차문은 안 열리고
차키라는 API로 열라는 요청을 해야 문이 열리듯이

프론트엔드의 요청을 API라는 공통된 통로를 이용해 정해진 규칙으로 백엔드에게
보내고 백엔드는 그걸 실행하는 구조로 프론트엔드와 백엔드는 서로 상호작용 됩니다.
